# Phase 2 R&D Orientation — zcrypto

*Consolidates the two Phase 2 research reports (strategy/methodology + on-chain data) into
one orientation. Companion to [`01.binance-eea-spot-quant.md`](01.binance-eea-spot-quant.md)
(original roadmap) and [`02.phase1-summary.md`](02.phase1-summary.md) (Phase 1 verdict).
Written to be critical, not confirmatory: the goal is to find a genuine edge cheaply — or to
prove cheaply that none beyond beta-timing exists. The open-topics it spawns are `T0019`
(trend + vol-target core), `T0020` (BTC→alt lead-lag), and `T0021` (on-chain regime vs SMA).*

## 0. The one-paragraph orientation

Phase 1 did not find a weak edge that better tuning will strengthen. It found that the
**premise** (cross-sectional ML ranking on daily OHLCV) is structurally weak for this market,
*and* that the only thing that survived — a regime-gated inverse-vol majors basket at ~0.50
OOS Sharpe — is almost certainly **beta-timing, not alpha**, and at ~0.50 sits **below the
~1.0–1.6 net Sharpe a naive trend-following rule on majors earns in the literature**. So
Phase 2's real question is not "how do I improve the ranker." It is: **is there any capturable
edge beyond a simple trend + volatility-target rule on majors — and if so, in which of two
channels does it live?** The two reports converge on that answer, and the convergence is the
spine of this plan.

## 2. The organizing idea — two channels, only one of which on-chain can touch

**Channel A — Market-timing / regime ("when and how much exposure").** Where Phase 1's only
survivor lives. Candidates: trend-following variants, vol-targeting, derivatives-positioning
extremes (funding/OI/long-short), and **on-chain regime signals** (MVRV-Z, NUPL, stablecoin
dry-powder, exchange flows). On-chain is usable **only** here — as a BTC/ETH-level overlay.

**Channel B — Genuine relative/cross-sectional alpha ("which coin outperforms").** Harder,
less certain, the channel Phase 1 refuted for daily OHLCV. Candidates: **residual
(beta-neutral) momentum**, **BTC→alt lead-lag**, funding dispersion, intraday microstructure.
**On-chain does NOT help here** — PEPE/APT/ARB and young tokens have structurally meaningless
holder metrics.

The uncomfortable consequence: **on-chain, trend/vol-target, and derivatives signals all
compete for the *same* Channel-A timing job.** On-chain does not open a new alpha channel for
the alt basket — it is, at best, a *better regime gate* (prior: comparable to a 200-day SMA,
not better). That demotes on-chain from "the frontier" Phase 1 imagined to "one contender for
the timing overlay."

## 4. Data plan (free-first; paid only as contingency)

**Tier 1 — free Binance archive** (already partly wired): spot + futures **klines** (1m→1d)
for #1/#3/#4/#6; **aggTrades** (tick, taker side); futures **`metrics/`** (daily OI, top-trader
& global long-short, taker buy/sell ratio) + **funding/mark/index/premium** for #2. ⚠ **OI &
long-short are daily-granularity in the archive** — intraday positioning isn't free and only
accrues live going forward.

**Tier 2 — free on-chain stack** (covers the whole research phase at $0 for #5): **Coin Metrics
Community** (pre-computed BTC/ETH metrics, CSV, no key); **Google BigQuery public chains** (raw
BTC/ETH/LTC/DOGE/Polygon/XRP/Cardano…; 1 TB/mo free; compute any metric; **CC-BY,
offline-reusable**); **Flipside/Dune** free SQL for alt/DeFi.

**Tier 3 — one paid month, ONLY if Tier 2 has a proven gap** (usually *entity-adjusted*
exchange/holder-cohort data you can't reconstruct): **CryptoQuant Professional, $109 for one
month** — **not** the $39 Advanced, which **lacks the Data API** for a bulk pull (the
"export-tier trap," the main gotcha). Pull BTC/ETH/covered-alts flows, SSR, MVRV, SOPR, NUPL,
miner flows; cancel before renewal (access persists to billing date). Fits
subscribe→bulk-pull→cancel→resume; offline retention permitted for single-researcher use (read
the live ToS). Glassnode Advanced (~$39) is the sub-$50 alternative but means slow manual
exports, not API bulk — confirm full history is unlocked first.

**Do NOT** build per-alt on-chain holder features for PEPE/APT/ARB; treat any unpublished
vendor "signal" as marketing until self-validated.

## 5. Validation upgrade (fix the two Phase-1 diseases)

Phase 1 was inflated by **single-bear holdout** + **18-recipe selection bias (~+0.38)**.
Prevent both:

1. **Always benchmark vs the Stage-0 null in the same harness** so "edge beyond beta" is
   measured every run, not assumed.
1. **CPCV primary; report the full path distribution, never the best path.**
1. **Stationary-bootstrap CIs** on Sharpe (vary block size train vs test for OOD stress) —
   *mitigates, doesn't cure, the single-regime problem.*
1. **Deflated Sharpe with the *true* trial count** — pre-register a trial budget, **log every
   config as one trial** (Harvey-Liu-Zhu: real hurdle t > 3.0).
1. **Budget holdout "looks"** — reserve the true holdout, iterate on CPCV/bootstrap, each look
   burns a trial.

## 6. Staged sequence + success/kill bar

- **Stage 0 — Falsify the premise.** Build the null (200d-SMA-gated inverse-vol 10 majors);
  compute net Sharpe under CPCV + bootstrap CIs. Proceed only with candidates clearing its CI.
  **If nothing beats it: the honest deliverable is "this is beta-timing — deploy the simple
  rule or stop."**
- **Stage 1 — Highest-EV core.** Per-asset **TSMOM + vol-targeting** (#1) at 4h/1d, fed by
  **intraday realized-vol** (#4). Target **net Sharpe ≥ ~1.0** surviving bootstrap and beating
  Stage 0.
- **Stage 2 — New information.** (a) **BTC→alt lead-lag** (#3); (b) **derivatives-positioning**
  (#2); (c) **on-chain gate vs SMA** head-to-head (#5); (d) a **small residual cross-sectional
  tilt** (#6). Add each only if it beats Stage 1 on the **CPCV distribution**, not best-path.
- **Stage 3 — Conditional.** Only if a primary signal with high recall emerges: \*\*meta-labeling
  - regime-conditional ensembling\*\* (#7), triple-barrier labels.

**Permanent kill/shelve:** if **residualization** (#6) doesn't make cross-sectional
regime-robust under CPCV → **abandon cross-sectional ranking**; if **vol-targeted TSMOM** clears
~1.0 net and beats the null → it's the **core**, and DL (#9) + pairs (#8) are **shelved**; if
**funding/lead-lag** don't *stack* on trend → keep them as risk overlays only; if the **on-chain
gate** only ties the SMA → keep the SMA, don't pay to re-pull.

## 7. Caveats

- **The deepest risk: there may be no capturable alpha beyond beta-timing** for a $10k spot-only
  retail account here. The honest outcome may be a simple trend + vol-target rule at
  ~beta-plus. The literature's net crypto-trend ceiling (~1.0–1.6) is *modest*.
- **Every Sharpe above ~1.5 cited is gross / often in-sample / short bull-heavy windows.** Apply
  12 bps round-trip + selection discount first. (Crypto carry: 6.4 full-sample → ~4 in 2024 →
  negative 2025 — the cautionary archetype.)
- **On-chain is a BTC/ETH regime overlay, not alt cross-sectional alpha**, prior vs the SMA =
  comparable; the decisive test is unpublished — assume nothing.
- **Bootstrap mitigates but doesn't cure the single-regime problem** — a new regime (high-rate
  macro, or an alt-season melt-up) can still break 2020–2026-fit conclusions.
- **Commercial track records are marketing** — upper bounds.
- **Pricing/tiers are mid-2026 snapshots** — confirm at checkout (esp. the CryptoQuant
  Advanced-vs-Professional API trap).
